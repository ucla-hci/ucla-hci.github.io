# course website

## requirements
- generalizable to different courses; actual contents of a course can imported, e.g., as a .yml file
- easy to update: for the same course to be offered in a new quarter, we can easily run a command to update the syllabus to the new calendar
- ideally everything within one page

## existing course websites i used
i tried notion (https://uclahci.notion.site/2026-Winter-ECE-CS-M119-Fundamentals-of-Networked-Embedded-Systems-29c0b2facd2981749752f32edaee27cb?source=copy_link) and spreadsheet (https://docs.google.com/spreadsheets/d/1EPz30ZUIgdEZhgZbZs4WsEvTqstY9_ljpiD0pX9Juh4/edit?gid=0#gid=0)--these two examples give you an idea what are the elements a course website should include

notion looks good but
- hard to separate data from ui
- hard to update new dates for the same course

spreadsheet is flexible but
- too unlike a website
- the cell grid limits how information is presented
